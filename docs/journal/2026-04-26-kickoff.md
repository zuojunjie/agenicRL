## 第二波执行（continuation）

### 精读两个核心文件（12:18–12:25）

读完 `external/Search-R1/infer.py` 全 135 行 + `search_r1/search/retrieval_server.py` 全 393 行。

笔记沉淀于 `notes/agent-loop.md`，关键发现：

- **两进程架构**：Policy LLM 进程 ←HTTP→ Retrieval Server 进程，始终独立
- **StopOnSequence 是工具调用边界的硬编码**：列举 `</search>` 各种空格/换行变体，防止模型自我幻想 `<information>`
- **retrieval_server.py 写死了 CUDA**（`model.cuda()` × 3 处），Mac 上跑真服务器需要 patch；本地烟雾测试 mock 即可
- **训练时 reward = exact_match / F1**，**`<information>` token 必须 mask 掉**（否则模型会去模仿检索结果）
- 与 GRPO 的连接：**Search-R1 = GRPO + 多轮工具调用 + outcome reward + token mask**

### 烟雾测试通过（12:23）

`scripts/smoke_test.py` 编写完成，mock 掉 `search()`，跑 Qwen2.5-0.5B-Instruct on MPS：
